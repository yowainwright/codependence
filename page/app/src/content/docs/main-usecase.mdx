## Main Usecase

Pin what you want, update the rest—unlike `npm update` or `yarn upgrade` which update all or nothing.

## Key Features

- **Version pinning:** Use `~` or `^` for precise control
- **Package manager agnostic:** Works with npm, yarn, pnpm, bun
- **Monorepo support:** Single command across multiple packages

## Why This Matters

- Check/update the dependencies you list
- Optionally pin critical dependencies and auto-update everything else with `--permissive`
- Maintain consistency across packages
- Automate in CI/CD
